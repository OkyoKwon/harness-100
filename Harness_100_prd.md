# Harness 100 UI — Product Requirements Document

## 1. 개요

### 1.1 제품 비전

**Harness 100 UI**는 100개의 에이전트 팀 하네스를 시각적으로 탐색, 커스터마이징, 프로젝트에 바로 적용할 수 있는 **완전 개방형** 웹 도구이다. 로그인, 회원가입, 결제 없이 누구나 접속 즉시 하네스를 찾고, 수정하고, 내 프로젝트에 세팅하거나 ZIP으로 내려받을 수 있다. "열고 → 고르고 → 세팅" 3단계로 끝나는 경험을 목표로 한다.

### 1.2 문제 정의

| 현재 Pain Point | 영향 |
|---|---|
| 100개 하네스를 텍스트 목록에서 탐색해야 함 | 원하는 하네스를 찾기 어려움 |
| `cp -r` 명령어로 수동 복사 필요 | 비개발자 접근성 낮음 |
| 에이전트/스킬 내용을 직접 .md 파일 열어야 확인 가능 | 하네스 구성 파악에 시간 소모 |
| 에이전트 커스터마이징 시 마크다운 직접 편집 필요 | 오류 가능성 높음, 포맷 불일치 위험 |
| 여러 하네스를 조합하려면 수동 병합 필요 | 복합 워크플로우 구성 진입장벽 |

### 1.3 목표 사용자

| 페르소나 | 설명 | 핵심 니즈 |
|---|---|---|
| **콘텐츠 크리에이터** | 유튜브, 뉴스레터, SNS 운영자 | 콘텐츠 관련 하네스를 빠르게 찾아 적용 |
| **소프트웨어 개발자** | 풀스택/DevOps 엔지니어 | 개발 워크플로우 자동화, 커스터마이징 |
| **비즈니스 기획자** | PM, 스타트업 대표, 전략 담당 | 전략/비즈니스 하네스 조합 활용 |
| **교육/연구자** | 강사, 학생, 연구원 | 학습/연구 하네스 탐색 및 맞춤 설정 |

### 1.4 핵심 설계 원칙

| 원칙 | 설명 |
|---|---|
| **제로 마찰 (Zero Friction)** | 로그인, 회원가입, 결제 일체 없음. URL 접속 즉시 모든 기능 사용 가능 |
| **3클릭 완료** | 카테고리 선택 → 하네스 클릭 → 세팅/다운로드. 최대 3클릭으로 목표 달성 |
| **상태 비저장 (Stateless)** | 서버에 사용자 데이터를 저장하지 않음. 커스터마이징은 브라우저 로컬에서 처리 |
| **모바일 우선** | 이동 중에도 하네스를 탐색하고 공유 링크를 받을 수 있도록 반응형 |

### 1.5 성공 지표

| 지표 | 목표 |
|---|---|
| 첫 방문 → 세팅/다운로드 완료 시간 | < 1분 (현재 ~10분) |
| 첫 방문 이탈률 (Bounce Rate) | < 30% |
| 월간 방문자 (MAV) | 출시 3개월 후 10,000+ |
| 하네스 획득 전환율 (세팅 + 다운로드) | > 40% |
| 세팅 비율 (획득 중 세팅 사용) | > 60% (ZIP 대비 세팅 선호 유도) |
| 커스터마이징 후 획득 비율 | > 20% |

## 2. 핵심 기능

- **정렬**: 번호순, 에이전트 수 순, 카테고리 순
- **카드 액션**: ☆ 즐겨찾기, [세팅 →], [ZIP ↓] — 상세 안 봐도 바로 사용 가능

### 2.3 즐겨찾기

로그인 없이 하네스를 즐겨찾기할 수 있다. localStorage 기반으로 브라우저에 저장.

#### 2.3.1 즐겨찾기 토글

카탈로그 카드와 상세 페이지 모든 곳에서 하트/별 아이콘 클릭으로 즉시 토글.

```
┌──────────┐
│ 01    ★  │  ← 즐겨찾기 토글 (채워진 별 = 추가됨)
│ YouTube  │
│ Prod.    │
│ 👥 5명   │
│ [다운로드]│
└──────────┘
```

#### 2.3.2 즐겨찾기 탭

카테고리 탭 맨 앞에 "즐겨찾기" 탭이 추가된다. 즐겨찾기한 하네스만 모아볼 수 있다.

- 즐겨찾기한 하네스를 한 번에 로컬 디렉토리에 세팅 (일괄 설치)
- localStorage에 ID 배열로 저장 → 같은 브라우저에서 재방문 시 유지
- 즐겨찾기 목록을 URL로 공유 가능 (예: `?favorites=01,16,43`)

### 2.4 다운로드 & 로컬 세팅

하네스를 내 프로젝트에 적용하는 2가지 방법을 제공한다.

#### 2.4.1 ZIP 다운로드 (기본)

별도 설치 마법사 없음. 1클릭 ZIP.

| 방식 | 동작 | 사용자 |
|---|---|---|
| **ZIP 다운로드** (기본) | 브라우저에서 JSZip으로 생성, 즉시 다운로드 | 모든 사용자 |
| **CLI 명령 복사** | `cp -r` 명령어를 클립보드에 복사 (보조) | CLI 선호 개발자 |

**다운로드 후 안내 (인라인 배너):**
- 폴더 구조 시각적 가이드 (어디에 `.claude/` 폴더를 넣어야 하는지)
- Claude Code 실행 명령어 (클립보드 복사 버튼 포함)
- 같은 카테고리의 관련 하네스 추천 (3개)

#### 2.4.2 로컬 디렉토리 직접 세팅

```
    │
    ▼
```

| 항목 | 상세 |
|---|---|
| **API** | [File System Access API](https://developer.mozilla.org/en-US/docs/Web/API/File_System_API) (W3C) |
| **지원 브라우저** | Chrome 86+, Edge 86+, Opera 72+ |
| **미지원 브라우저 폴백** | Safari, Firefox → 자동으로 ZIP 다운로드로 전환 |
| **권한** | 사용자가 폴더를 선택하고 명시적으로 쓰기 권한을 허용해야 함 |
| **보안** | 사용자가 직접 선택한 폴더에만 쓸 수 있음. 임의 경로 접근 불가 |

**브라우저 미지원 시 자동 폴백:**

```
[세팅 →] 클릭
    │
    ├── Chrome/Edge? → File System Access API로 직접 세팅
    │
    └── Safari/Firefox? → "이 브라우저에서는 ZIP 다운로드로 대체됩니다"
                          → ZIP 자동 다운로드 + 수동 배치 안내
```

## 4. 기술 아키텍처

### 4.1 시스템 구성

서버 없이 정적 사이트로 배포한다. 별도 백엔드, DB, 인증 시스템이 필요 없다.

```
┌──────────────────────────────────────────────────────┐
│               정적 SPA (Static Site)                  │
│  Next.js 15 (Static Export) + React 19 + TypeScript  │
│  Tailwind CSS + shadcn/ui + React Flow               │
└─────────────────────┬────────────────────────────────┘
                      │
          빌드 시 GitHub 저장소에서
          .md 파일 파싱 → JSON 생성
                      │
┌─────────────────────▼────────────────────────────────┐
│              정적 호스팅 (Vercel / Cloudflare Pages)   │
│  ┌─────────────┐  ┌──────────────────────────────┐  │
│  │ /data/*.json │  │ 클라이언트 사이드              │  │
│  │ (하네스 전체 │  │ - 검색: Fuse.js (브라우저)    │  │
│  │  메타데이터)  │  │ - 즐겨찾기: localStorage       │  │
│  └─────────────┘  │ - 커스터마이징: localStorage   │  │
│                    │ - ZIP: JSZip (브라우저)        │  │
│                    │ - 세팅: File System Access API │  │
│                    └──────────────────────────────┘  │
└──────────────────────────────────────────────────────┘
```

**핵심: 모든 로직이 브라우저에서 실행된다.**

- 검색/필터: 빌드 시 생성된 JSON을 Fuse.js로 클라이언트 검색
- 즐겨찾기: localStorage에 ID 배열 저장, URL 쿼리로 공유 가능
- 커스터마이징: localStorage에 임시 저장, 브라우저 닫으면 초기화 (의도적 설계)
- 다운로드: JSZip으로 브라우저에서 직접 ZIP 생성 후 다운로드
- 로컬 세팅: File System Access API로 브라우저에서 직접 폴더에 쓰기 (Chrome/Edge)
- 배포: `next export`로 정적 파일만 생성, 서버 비용 0원

### 4.2 핵심 데이터 모델

```typescript
// 하네스 메타데이터 (GitHub 저장소에서 파싱)
interface Harness {
  readonly id: number;             // 01-100
  readonly slug: string;           // "youtube-production"
  readonly name: string;           // "YouTube Production"
  readonly description: string;
  readonly category: Category;
  readonly agents: ReadonlyArray<Agent>;
  readonly skill: Skill;
  readonly frameworks: ReadonlyArray<string>;
  readonly agentCount: number;     // 4 | 5
}

interface Agent {
  readonly id: string;
  readonly name: string;
  readonly role: string;
  readonly description: string;
  readonly tools: ReadonlyArray<string>;
  readonly outputTemplate: string;
  readonly dependencies: ReadonlyArray<string>;  // 다른 에이전트 ID
}

interface Skill {
  readonly id: string;
  readonly name: string;
  readonly triggerConditions: ReadonlyArray<string>;
  readonly executionOrder: ReadonlyArray<ExecutionStep>;
  readonly modes: {
    readonly full: ExecutionPlan;
    readonly reduced: ExecutionPlan;
    readonly single: ExecutionPlan;
  };
}

// 커스터마이징 상태 (localStorage에 저장, 서버 전송 없음)
interface LocalCustomization {
  readonly baseHarnessId: number;
  readonly modifications: ReadonlyArray<Modification>;
  readonly createdAt: string;      // ISO 8601
}

// 즐겨찾기 (localStorage에 저장)
// key: "harness100_favorites", value: JSON.stringify(number[])
// 예: [1, 16, 43]
type Favorites = ReadonlyArray<number>;

// 최근 세팅한 프로젝트 경로 (localStorage에 저장, 재사용 편의)
// key: "harness100_recent_paths"
interface RecentPath {
  readonly path: string;           // "/Users/me/my-project"
  readonly lastUsed: string;       // ISO 8601
  readonly harnessIds: ReadonlyArray<number>;  // 이 경로에 세팅한 하네스들
}

interface ExecutionStep {
  readonly agentId: string;
  readonly parallel: boolean;        // true면 이전 단계와 병렬 실행
  readonly dependsOn: ReadonlyArray<string>;  // 선행 에이전트 ID
}

type ExecutionPlan = ReadonlyArray<ExecutionStep>;

interface Modification {
  readonly agentId: string;
  readonly field: "name" | "role" | "description" | "outputTemplate" | "enabled";
  readonly value: string | boolean;
}

type Category =
  | "content"
  | "development"
  | "data-ai"
  | "business"
  | "education"
  | "legal"
  | "lifestyle"
  | "communication"
  | "operations"
  | "specialized";
```

### 4.3 빌드 타임 파싱 파이프라인

GitHub 저장소의 마크다운 파일을 **빌드 시** 정적 JSON으로 변환한다. 런타임 서버가 필요 없다.

```
[빌드 시 (CI/CD)]

GitHub Repo (.md files)
    │
    ▼
[1] Clone (git submodule 또는 GitHub API)
    │
    ▼
[2] Parse CLAUDE.md → Harness metadata
    │
    ▼
[3] Parse agents/*.md → Agent definitions
    │
    ▼
[4] Parse skills/*/skill.md → Skill definitions
    │
    ▼
[5] Generate /public/data/harnesses.json (전체 카탈로그)
    Generate /public/data/harness-{id}.json (개별 상세)
    │
    ▼
[6] Static export → Vercel / Cloudflare Pages 배포
```

**저장소 업데이트 반영**: GitHub Webhook → Vercel 재빌드 트리거 (자동)

## 6. 사용자 시나리오

### 6.0 유저 온보딩 — "열고 고르고 받기"

**설계 철학**: 로그인, 회원가입, 프로필 설정 일체 없음. URL을 열면 즉시 하네스 카탈로그가 보이고, 클릭 몇 번이면 다운로드까지 끝난다.

#### 전체 여정 한눈에 보기

```
접속 ──→ 탐색 ──→ ★ 즐겨찾기 ──→ 확인 ──→ 세팅/다운로드
(0초)   (즉시)    (선택)        (1클릭)    (1클릭)

별도 단계 없음. 홈 = 카탈로그. 바로 시작.
즐겨찾기는 선택 사항 — 바로 세팅/다운로드해도 됨.
```

#### Step 1: 접속 즉시 카탈로그 (홈 = 카탈로그)

별도 랜딩 페이지 없음. URL을 열면 바로 100개 하네스 카탈로그가 노출된다.


**핵심 설계:**

- 홈 = 카탈로그. 랜딩 페이지 없음, 스플래시 없음, 로그인 모달 없음
- 카테고리 탭 맨 앞에 **★ 즐겨찾기** 탭 → 내가 찜한 하네스만 모아보기
- 카테고리 탭으로 즉시 필터링 (10개 카테고리 수평 스크롤)
- 검색창은 항상 상단 고정 (Fuse.js 클라이언트 사이드 검색)
- 각 카드에 **☆ 즐겨찾기** 아이콘 → 클릭 한 번으로 즐겨찾기 추가/제거
- 각 카드에 **[다운로드]** + **[세팅 →]** 버튼 2개 노출
  - [다운로드]: ZIP 파일로 받기
  - [세팅 →]: 로컬 프로젝트 폴더에 바로 설치 (File System Access API)
- 카드 클릭 시 상세 페이지 진입 (더 알고 싶을 때만)

#### 온보딩 전체 퍼널

```
접속 ──→ 탐색 ──→ 즐겨찾기/관심 ──→ 세팅 or ZIP ──→ 재방문
100%      95%        70%              55%            30%
       (홈=카탈로그) (☆ 즐겨찾기 +    ([세팅→]로     (같은 프로젝트에
                     카드 클릭)       폴더 선택 1번)  하네스 추가)
```

| 퍼널 단계 | 전환 목표 | 핵심 장치 |
|---|---|---|
| 접속 → 탐색 | 95% | 홈이 곧 카탈로그, 별도 단계 0개 |
| 탐색 → 관심 | 70% | ☆ 즐겨찾기로 찜, 카드에 핵심 정보 즉시 노출 |
| 관심 → 세팅/다운로드 | 55% | [세팅 →]로 폴더 선택 1번이면 끝. ZIP도 1클릭 |
| 세팅 → 재방문 | 30% | "다른 하네스도 이 프로젝트에 추가" 유도, 즐겨찾기 목록에서 추가 세팅 |

### 시나리오 1: 가장 빠른 경로 — 검색 → 바로 세팅 (30초)

> **사용자**: 이미 원하는 하네스를 아는 개발자, 지금 작업 중인 프로젝트가 있음

1. 접속 → 검색창에 "CI/CD" 입력
2. `20 - CI/CD Pipeline` 카드의 **[세팅 →]** 클릭
3. 폴더 선택 대화상자에서 프로젝트 폴더 선택
4. `.claude/` 폴더 자동 생성 → 완료 안내 → 바로 `claude` 실행
5. 끝. 상세 페이지도 안 열었음. ZIP 압축 해제도 없음.

### 시나리오 2: 탐색 → 즐겨찾기 → 나중에 세팅 (1분 + α)

> **사용자**: 유튜브를 시작하려는 크리에이터, 여러 하네스를 비교하고 싶음

1. 접속 → "콘텐츠" 카테고리 탭 클릭 → 15개 하네스 노출
2. `01 - YouTube Production` 카드의 ☆ 클릭 → ★ 즐겨찾기 추가
3. `10 - Social Media Manager`도 ☆ 클릭 → ★ 즐겨찾기 추가
4. 각 상세 페이지를 둘러보며 에이전트 구성 비교
5. "★ 즐겨찾기" 탭 클릭 → 찜한 2개 하네스만 모아보기
6. 마음이 정해지면 **[세팅 →]** 클릭 → 프로젝트에 바로 적용

### 시나리오 3: 커스터마이징 → 세팅 (3분)

> **사용자**: 기술 블로그를 운영하는 개발자, 뉴스레터 에이전트를 자기 스타일로 수정하고 싶음

1. 접속 → 검색 "뉴스레터" → `03 - Newsletter Engine` 상세 진입
2. [수정해서 받기] 클릭 → 인라인 편집 모드
3. "큐레이터" 에이전트의 역할을 "기술 트렌드 전문 큐레이터"로 수정
4. "A/B 테스트 전문가" 에이전트를 체크 해제 (불필요)
5. **[수정본 세팅 →]** 클릭 → 프로젝트 폴더 선택 → 커스터마이징된 버전이 바로 설치됨

### 시나리오 4: 즐겨찾기 일괄 세팅 (2분)

> **사용자**: 스타트업 대표, 여러 하네스를 하나의 프로젝트에 한 번에 설치하고 싶음

1. 카탈로그에서 필요한 하네스를 즐겨찾기: `43`, `44`, `53`
2. "★ 즐겨찾기" 탭 → 3개 하네스 확인
3. **[즐겨찾기 전체 세팅 →]** 클릭 → 프로젝트 폴더 선택
4. 3개 하네스의 에이전트와 스킬이 `.claude/` 폴더에 병합 설치
5. 완료. 하나의 프로젝트에서 3개 하네스 스킬 모두 사용 가능

### 시나리오 5: 하네스 조합 → 세팅 (5분)

> **사용자**: 에이전트 단위로 직접 골라서 커스텀 팀을 만들고 싶은 고급 사용자

1. 상단 네비게이션에서 [조합기] 진입
2. 왼쪽 패널에서 에이전트 검색: "시장분석", "재무", "보고서"
3. 필요한 에이전트를 오른쪽 캔버스에 드래그 앤 드롭
4. 자동으로 의존관계 연결 → 워크플로우 확인
5. **[조합 세팅 →]** 클릭 → 프로젝트 폴더 선택 → 즉시 설치

### 시나리오 6: Safari 사용자 — ZIP 폴백 (1분)

> **사용자**: Safari 브라우저 사용자 (File System Access API 미지원)

1. 접속 → 하네스 카드의 **[세팅 →]** 클릭
2. "이 브라우저에서는 ZIP 다운로드로 대체됩니다" 안내 표시
3. ZIP 자동 다운로드 → 수동으로 프로젝트에 `.claude/` 폴더 배치
4. 사용 가이드 배너의 안내에 따라 Claude Code 실행

## 8. 릴리스 로드맵

### Phase 1: MVP (4주)

**목표**: 접속 → 탐색 → 즐겨찾기 → 다운로드/세팅 완성

- 정적 사이트 빌드 파이프라인 (GitHub → JSON → Static Export)
- 하네스 카탈로그 (카드 뷰 + 검색 + 카테고리 탭)
- 하네스 상세 페이지 (에이전트 구성 + 워크플로우 다이어그램)
- 즐겨찾기 (localStorage + URL 공유)
- 1클릭 ZIP 다운로드 (JSZip, 브라우저 생성)
- 로컬 디렉토리 직접 세팅 (File System Access API + Safari/Firefox ZIP 폴백)
- CLI 명령 클립보드 복사
- 다운로드/세팅 후 사용 가이드 배너
- 반응형 웹 (모바일 대응)
- Vercel 배포

### Phase 2: 인라인 커스터마이징 (4주)

**목표**: 수정해서 받기/세팅 완성

- 인라인 에이전트 편집기 (이름, 역할, 산출물 폼 편집)
- 에이전트 포함/제외 체크박스
- 수정본 ZIP 다운로드 또는 로컬 세팅 (수정 내용 반영)
- 원본 복원 기능
- localStorage 임시 저장 (같은 세션 내 편집 내용 유지)
- 즐겨찾기 일괄 다운로드/세팅 (여러 하네스 병합)

### Phase 3: 하네스 조합기 (4주)

**목표**: 여러 하네스의 에이전트를 드래그 앤 드롭으로 조합

- 하네스 조합기 (Mix & Match) 페이지
- 에이전트 검색 + 드래그 앤 드롭
- 의존관계 자동 연결
- 조합 결과 ZIP 다운로드 또는 로컬 세팅
- URL로 조합 공유 (쿼리 파라미터 인코딩, 서버 불필요)

### Phase 4: 고도화 (지속)

- 최근 세팅한 프로젝트 경로 기억 → 같은 프로젝트에 추가 세팅 시 원클릭
- 하네스 원본 저장소 업데이트 시 자동 재빌드
- 다크 모드
- 다국어 (영어) 지원
- 하네스 간 비교 뷰 (side-by-side)
- PWA 지원 (오프라인에서도 카탈로그 탐색)

### 9.2 하네스 품질 기준 체크리스트

모든 하네스(원본 + 커스텀)는 UI에서 다음 기준 충족 여부를 표시:

- [ ] 에이전트 팀 모드: SendMessage 직접 통신, 교차 검증
- [ ] 도메인 전문성: 실전 프레임워크 적용
- [ ] 산출물 템플릿: 구조화된 에이전트별 포맷
- [ ] 의존 관계 관리: 작업 순서 및 병렬 실행 명시
- [ ] 에러 핸들링: 실패 시 폴백 전략 정의
- [ ] 작업 규모별 모드: 풀/축소/단일 모드 지원
- [ ] 테스트 시나리오: 정상/기존파일/에러 흐름 3종
- [ ] 트리거 경계: should-trigger / NOT-trigger 명시

### 9.3 용어 사전

| 용어 | 정의 |
|---|---|
| **하네스 (Harness)** | 4~5명의 전문 에이전트와 오케스트레이터 스킬로 구성된 워크플로우 패키지 |
| **에이전트 (Agent)** | 특정 역할과 전문성을 가진 AI 에이전트 정의 (.md 파일) |
| **스킬 (Skill)** | 에이전트들의 실행 순서와 조건을 정의하는 오케스트레이터 |
| **트리거 경계** | 스킬이 자동 실행되어야 하는 조건(should-trigger)과 실행하지 말아야 하는 조건(NOT-trigger) |
| **작업 모드** | 풀(전체 에이전트 실행), 축소(핵심만), 단일(단독 에이전트) 모드 |
| **하네스 조합** | 서로 다른 하네스의 에이전트를 결합하여 새로운 워크플로우를 만드는 기능 |
| **세팅** | File System Access API를 이용해 브라우저에서 로컬 프로젝트 폴더에 .claude/ 를 직접 생성하는 기능 |
| **즐겨찾기** | localStorage에 저장되는 하네스 북마크. URL 파라미터로 공유 가능 |

#### 10.6.1 빌드 시 마크다운 파싱 (parse-harnesses.ts)

```typescript
// scripts/parse-harnesses.ts — 빌드 시 실행
// 입력: harness-100/{NN}-{name}/.claude/ 폴더들
// 출력: public/data/harnesses.json + public/data/harness/{id}.json

import matter from "gray-matter";
import { readFileSync, readdirSync, writeFileSync, mkdirSync } from "fs";

function parseHarness(dir: string): Harness {
  const claudeMd = readFileSync(`${dir}/.claude/CLAUDE.md`, "utf-8");
  const { data, content } = matter(claudeMd);

  const agentFiles = readdirSync(`${dir}/.claude/agents/`);
  const agents = agentFiles.map((file) => parseAgent(`${dir}/.claude/agents/${file}`));

  const skillDirs = readdirSync(`${dir}/.claude/skills/`);
  const skill = parseSkill(`${dir}/.claude/skills/${skillDirs[0]}/skill.md`);

  return { ...data, agents, skill, content } as Harness;
}
```

#### 10.6.2 File System Access API 래퍼 (local-writer.ts)

```typescript
// src/lib/local-writer.ts

interface SetupResult {
  readonly success: boolean;
  readonly filesWritten: number;
  readonly path: string;
  readonly error?: string;
}

async function setupToLocal(
  harness: Harness,
  modifications?: ReadonlyArray<Modification>
): Promise<SetupResult> {
  // 1. 브라우저 지원 확인
  if (!("showDirectoryPicker" in window)) {
    return { success: false, filesWritten: 0, path: "", error: "unsupported" };
  }

  // 2. 폴더 선택
  const dirHandle = await window.showDirectoryPicker({ mode: "readwrite" });

  // 3. .claude/ 폴더 생성 (또는 기존 확인)
  const claudeDir = await dirHandle.getDirectoryHandle(".claude", { create: true });

  // 4. CLAUDE.md 쓰기
  const claudeMdFile = await claudeDir.getFileHandle("CLAUDE.md", { create: true });
  const claudeMdWriter = await claudeMdFile.createWritable();
  await claudeMdWriter.write(generateClaudeMd(harness));
  await claudeMdWriter.close();

  // 5. agents/ 폴더 + 각 에이전트 .md 쓰기
  const agentsDir = await claudeDir.getDirectoryHandle("agents", { create: true });
  for (const agent of applyModifications(harness.agents, modifications)) {
    const file = await agentsDir.getFileHandle(`${agent.id}.md`, { create: true });
    const writer = await file.createWritable();
    await writer.write(generateAgentMd(agent));
    await writer.close();
  }

  // 6. skills/ 폴더 + skill.md 쓰기
  // ... (동일 패턴)

  return { success: true, filesWritten: countFiles(harness), path: dirHandle.name };
}
```

#### 10.6.3 즐겨찾기 훅 (use-favorites.ts)

```typescript
// src/hooks/use-favorites.ts

const STORAGE_KEY = "harness100_favorites";

function useFavorites() {
  const [favorites, setFavorites] = useState<ReadonlyArray<number>>(() => {
    // URL 쿼리에서 먼저 확인 (공유 링크)
    const params = new URLSearchParams(window.location.search);
    const fromUrl = params.get("favorites");
    if (fromUrl) return fromUrl.split(",").map(Number);

    // localStorage에서 로드
    const stored = localStorage.getItem(STORAGE_KEY);
    return stored ? JSON.parse(stored) : [];
  });

  const toggle = useCallback((id: number) => {
    setFavorites((prev) => {
      const next = prev.includes(id)
        ? prev.filter((f) => f !== id)
        : [...prev, id];
      localStorage.setItem(STORAGE_KEY, JSON.stringify(next));
      return next;
    });
  }, []);

  const shareUrl = useMemo(() => {
    if (favorites.length === 0) return "";
    return `${window.location.origin}?favorites=${favorites.join(",")}`;
  }, [favorites]);

  return { favorites, toggle, isFavorite: (id: number) => favorites.includes(id), shareUrl };
}
```
